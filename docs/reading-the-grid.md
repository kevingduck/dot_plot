# Reading the grid

One row per user, one column per day, weekends shaded.

## Cells

- The **symbol** in a cell is the day's *rarest* event — so low-frequency,
  high-signal moments (created a playlist, exported a file) stay visible on
  days the everyday core event also happened.
- **Bigger marks** mean more events that day; the **mini dots** under a mark
  are the other event types that also happened.
- A **ring** marks each user's first day — scan down the ring diagonal to
  see how onboarding cohorts develop.
- Hover any cell for the exact per-event counts. Arrow keys move around the
  grid. Click a row for that user's full event log and streak stats.

## Legend, filters, sorting

- Click legend items to hide/show event types. The event marked **core** is
  your product's main value moment.
- Filter by date range (or a custom from/to), platform, plan, or user
  search; sort rows by signup date, most active days, recency, or longest
  streak.

## Cohort retention

Below the grid: the share of each signup-week cohort still active 1, 2, 3…
weeks after signup, scoped to the same filters. Only weeks the whole cohort
has fully lived through are charted — partial weeks would create fake
spikes.

## Insights

**✨ Find patterns** sends a compact per-user usage summary to Claude and
returns 3–5 ranked, concrete observations — churn risks, activation
hypotheses, users worth a customer interview. Click a card to highlight the
users it's about. Costs a few cents on the default model.
